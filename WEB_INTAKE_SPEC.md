## Purpose
Defines the logical flow for audit intake via a gated web interface.

## Flow
1. Client submits intake questionnaire
2. Tier selection recorded
3. Scope reviewed by HHI
4. SOW issued
5. Audit initiated upon execution

## Data Handling
- Intake data handled confidentially
- No training use of client data
- No third-party sharing

This specification does not include UI or implementation code.
